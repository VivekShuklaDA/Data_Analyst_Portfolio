Technologies Used

Nashville data cleaning

Project Overview

Executed a complete data cleaning pipeline on the Nashville Housing dataset using SQL Server. The objective was to transform raw, inconsistent real estate transaction data into a clean, structured format suitable for analysis. The project involved standardizing date formats, handling null values, splitting complex address fields, correcting categorical inconsistencies, and removing duplicate records. A new cleaned dataset was created and populated with enhanced data for further use in analysis or dashboarding.

Technologies Used

SQL Server Data Cleaning CTEs Window Functions String Functions Data Transformation Conditional Logic

Key Features

  • Created a new NashvilleHousingCleaned table structure using SELECT INTO
  • Standardized inconsistent SaleDate formats using CONVERT() and added a new formatted column
  • Filled missing PropertyAddress values by joining on ParcelID and using ISNULL()
  • Split PropertyAddress into separate columns: Street and City using SUBSTRING() and CHARINDEX()
  • Parsed OwnerAddress into Address, City, and State using PARSENAME() and REPLACE()
  • Standardized SoldAsVacant values from 'Y'/'N' to 'Yes'/'No' using a CASE statement
  • Identified and removed duplicate records using a CTE with ROW_NUMBER()
  • Dropped temporary or unnecessary columns after transformations were complete
  • Inserted cleaned and transformed data into the NashvilleHousingCleaned table for future analysis
